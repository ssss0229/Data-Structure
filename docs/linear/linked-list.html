연결 리스트Linked List

연결 리스트(Linked List)는 데이터를 담는 노드(Node)가 포인터를 통해 연결된 선형 자료구조입니다.

각 노드는 데이터와 다음 노드의 주소(혹은 이전 노드의 주소)를 가지고 있으며, 자료 접근은 순차적으로 이루어집니다.

연결 리스트의 구조

배열과 달리 메모리 상에서 노드는 연속적이지 않고 흩어져 저장됩니다.

각 노드는 다음 노드를 가리키는 포인터를 가지고 있어, 포인터를 따라가며 순차적으로 접근할 수 있습니다.

장단점

장점

  • 데이터 삽입/삭제가 유연하며 중간 위치에서도 가능
  • 메모리를 효율적으로 사용 가능
  • 배열처럼 미리 크기를 지정할 필요 없음

단점

  • 인덱스 접근 불가, 탐색은 순차적이어야 함 → O(n)
  • 포인터를 사용하므로 구현과 이해가 배열보다 복잡
  • 추가적인 포인터 메모리 사용

연결 리스트의 종류

  • 단일 연결 리스트(Singly Linked List) : 각 노드가 다음 노드만 가리킴
  • 이중 연결 리스트(Doubly Linked List) : 각 노드가 이전과 다음 노드를 모두 가리킴
  • 원형 연결 리스트(Circular Linked List) : 마지막 노드가 첫 번째 노드를 가리켜 순환 구조

구현 코드 및 연산 설명 (Python)


class Node:
    def __init__(self, data):
        self.data = data
        self.next = None

class LinkedList:
    def __init__(self):
        self.head = None

    def append(self, data):
        new_node = Node(data)
        if not self.head:
            self.head = new_node
            return
        current = self.head
        while current.next:
            current = current.next
        current.next = new_node

    def delete(self, data):
        current = self.head
        prev = None
        while current:
            if current.data == data:
                if prev:
                    prev.next = current.next
                else:
                    self.head = current.next
                return
            prev = current
            current = current.next

append : 맨 뒤에 노드 추가 → O(n)
delete : 특정 데이터 노드 삭제 → O(n)

연결 리스트 사용 시 주의점

  • null 포인터 확인 필수
  • 순환 구조 시 무한 루프 주의
  • 메모리 해제/누수 주의
