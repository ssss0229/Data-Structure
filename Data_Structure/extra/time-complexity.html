시간 복잡도

시간 복잡도란?

시간 복잡도(Time Complexity)는 알고리즘이 실행되는 데 걸리는 연산 횟수의 증가 추세를 입력 크기 n에 따라 표현한 것이다.

왜 실행 시간이 아니라 연산 횟수일까?

실제 실행 시간은 컴퓨터 성능, 언어, 환경에 따라 달라진다. 따라서 알고리즘 자체의 효율을 비교하기 위해 연산 횟수의 증가율을 기준으로 삼는다.

대표적인 시간 복잡도

  • O(1) : 한 번의 연산
  • O(log n) : 이진 탐색
  • O(n) : 선형 탐색
  • O(n log n) : 효율적인 정렬 알고리즘
  • O(n²) : 중첩 반복문

예시

배열에서 특정 값을 찾기 위해 처음부터 끝까지 확인하면 최악의 경우 O(n)의 시간 복잡도를 가진다.

정렬된 배열에서 이진 탐색을 사용하면 O(log n)으로 더 빠르게 탐색할 수 있다.

시간 복잡도를 분석하는 이유

입력 데이터가 커질수록 알고리즘의 차이는 극명해진다. 시간 복잡도 분석은 확장 가능한 프로그램을 설계하는 데 필수적이다.
