큐Queue

큐(Queue)는 데이터를 먼저 들어온 순서대로 처리하는 선형 자료구조입니다. FIFO(First In, First Out) 구조를 가지며, 삽입은 rear에서, 삭제는 front에서 이루어집니다.

배열이나 연결 리스트 기반으로 구현되며, 각 요소는 다음 노드/인덱스를 가리키는 포인터로 연결될 수 있습니다.

구현 방식

  • 배열 기반 큐 : front, rear 인덱스를 사용하여 구현
  • 연결 리스트 기반 큐 : 노드와 포인터로 front → rear 연결

장단점

장점

  • 데이터가 들어온 순서를 보장
  • 삽입/삭제 연산이 큐 끝에서 효율적
  • 연결 리스트 기반이라면 크기 제한 없음

단점

  • 중간 위치 접근 불가 → 순차 탐색 필요
  • 배열 기반 큐는 고정 크기 시 overflow 발생 가능
  • 포인터 사용으로 메모리 추가 소요

큐의 종류

  • 선형 큐(Linear Queue) : 일반적인 FIFO 큐
  • 원형 큐(Circular Queue) : 배열 끝과 시작을 연결하여 공간 효율화
  • 우선순위 큐(Priority Queue) : 우선순위에 따라 요소를 처리
  • 덱(Deque) : 양쪽(front/rear)에서 삽입/삭제 가능

구현 코드 및 연산 설명 (Python)


class Queue:
    def __init__(self):
        self.data = []

    def enqueue(self, value):
        self.data.append(value)  # rear에서 추가

    def dequeue(self):
        if self.is_empty():
            raise IndexError("Queue is empty")
        return self.data.pop(0)  # front에서 제거

    def peek(self):
        if self.is_empty():
            raise IndexError("Queue is empty")
        return self.data[0]

    def is_empty(self):
        return len(self.data) == 0

enqueue : rear에 데이터 추가 → O(1) (연결 리스트), O(1)~O(n) (배열 기반, shift 필요 시)
dequeue : front에서 데이터 제거 → O(1) (연결 리스트), O(n) (배열 기반)
peek : front 데이터 확인 → O(1)

큐 사용 시 주의점

  • 비어 있는 큐에서 dequeue 수행 시 오류 발생
  • 배열 기반 큐에서 overflow 발생 가능
  • 원형 큐 구현 시 front/rear 포인터 관리 주의
